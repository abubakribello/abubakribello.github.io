The situation.

Every new client at DigitalXshift triggered the same manual sequence. Someone had to create the Google Drive folder structure, set up a Notion project page, create a Slack channel, add the right team members, and send out welcome documents. It was repetitive, easy to get wrong, and took 60-120 minutes per new client.

For a fast-growing consultancy adding multiple new clients each month, this was burning 35-50 hours that should have gone to actual client work. The process also relied on whoever was available remembering to do each step, which meant occasional gaps and inconsistencies.

What I built.

I built an onboarding pipeline that triggers automatically the moment a new client deal closes and handles every setup step without anyone touching it.

  • Webhook trigger: The pipeline fires automatically when a new client deal is marked as closed. No manual start required.
  • Google Drive setup: The system creates the full folder structure automatically, including a main client folder and all required subfolders for projects, contracts, and invoices.
  • Notion project page: A Notion page is created with client details, package information, and timeline pre-filled from the deal data. The team can start working from it immediately.
  • Slack channel and team setup: A dedicated Slack channel is created for the client and all relevant team members are added automatically.
  • Welcome communications: Welcome email and onboarding documents are sent to the client automatically without anyone composing or triggering them manually.
  • Full audit trail: Every action is logged so the team can always verify what was set up and when.
